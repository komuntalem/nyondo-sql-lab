Explanation:  Using f-strings or template literals like `WHERE name = '${input}'` is
dangerous because the user's input gets pasted directly into the SQL string before the
database sees it, so the database can't distinguish between your code and the user's data
and will execute injected commands like `' OR 1=1--` as real SQL. With parameterized queries
using `?` placeholders, the SQL is sent to the database and compiled as a fixed template first,
then the user input is sent separately as plain data, locking the query structure so the input 
is only ever treated as a literal string to match against rather than executable code.
